# タスク完了時のチェックリスト

このプロジェクトは純粋なHTML/CSS/JavaScriptで構成されており、自動化されたビルドやテストプロセスはありません。

## コード変更後の確認事項

### 1. 手動テスト
- [ ] ブラウザでindex.htmlを開き、すべての機能が正常に動作することを確認
- [ ] 以下の機能を個別にテスト:
  - [ ] スタートボタンをクリックして問題が表示される
  - [ ] 音声読み上げが正常に機能する
  - [ ] 4択ボタンをクリックして正誤判定が正しく動作する
  - [ ] 正解音・不正解音が再生される
  - [ ] 設定変更（段の選択、問題数の変更）が反映される
  - [ ] ゲーム履歴が正しく表示・保存される
  - [ ] localStorageの読み書きが正常に機能する

### 2. ブラウザ互換性確認
- [ ] Chrome/Edgeでの動作確認
- [ ] Safari（macOS）での動作確認
- [ ] Firefox（オプション）での動作確認

### 3. 開発者ツールでの確認
- [ ] コンソールにエラーが出ていないか確認
- [ ] ネットワークタブで404エラーがないか確認（correct.mp3, incorrect.mp3）
- [ ] localStorageに設定とゲーム履歴が正しく保存されているか確認

### 4. コードレビュー
- [ ] コードがプロジェクトのスタイルガイドに従っているか
- [ ] 不要なコンソールログや開発用コードが残っていないか
- [ ] コメントが適切に記述されているか（日本語）
- [ ] 変数名や関数名が分かりやすいか

### 5. Git操作
- [ ] `git status` で変更内容を確認
- [ ] `git diff` で差分を確認
- [ ] Conventional Commitsに従ってコミットメッセージを作成
- [ ] 絵文字を含めたコミットメッセージを使用

## タスク完了の定義
- すべての機能が期待通りに動作する
- ブラウザコンソールにエラーがない
- コードがプロジェクトの規約に従っている
- 適切なコミットメッセージでコミットされている
